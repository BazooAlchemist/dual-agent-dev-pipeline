# 附录：文件触发样例

> **非规范性声明：** 本文档是非规范性附录，仅供参考。判定影响分级请使用 `core/04-dual-agent-audit.md` 中的能力触发器规则。

## 能力域与典型文件示例

下表列出每个能力域中常见文件的模式匹配示例。**文件举例不等于判定规则。** 文件发生了变更但所触及的能力域未被触发时，不应判定为对应影响级别。

| 能力域 | 影响级别 | 典型文件模式举例 | 说明 |
|--------|---------|----------------|------|
| 桥接协议 | 高 | `*bridge*.py`, `*worker*.mjs`, `bridge-protocol.*`, `adapters/*.md` | 变更可能影响跨 agent 通信协议 |
| 网关核心 | 高 | `gateway.*`, `openclaw.json`, `routing.*`, `*session*.*` | 调度或消息路由逻辑 |
| 安全与认证 | 高 | `*auth*.*`, `*token*.*`, `permissions.*`, `keys.*`, `*.pem`, `*.crt` | 凭据或访问控制 |
| 治理规则 | 高 | `*governance*.*`, `*audit*.*`, `*gate*.*`, `CLAUDE.md` | AI 行为约束与审查规则 |
| Hook 系统 | 中 | `hooks/*.js`, `*handler*.*`, `*hook*.*`, `events.*` | 事件处理逻辑变更 |
| 数据模型 | 中 | `schema.*`, `*model*.*`, `migrations/*`, `*sql` | 数据结构或持久化变更 |
| 外部集成 | 中 | `integrations/*`, `*client*.*`, `*api*.*`, `webhook.*` | 与外部系统对接 |
| 技能/工具 | 低 | `skills/*.md`, `tools/*`, `*command*.*` | AI 技能或工具定义 |
| 配置参数 | 低 | `settings.*`, `config.*`, `*.env`, `*.plist` | 非认证相关的运行参数 |
| 文档注释 | 低 | `*.md`, `*.rst`, `docs/*` | 纯文档或代码内注释 |

## 文件触发判定流程

1. 检查变更文件是否落入上表的文件模式
2. 确认变更是否触发了对应能力域的执行逻辑
3. 若能力域未触及（例如仅修改文档中的拼写错误），则不应用该能力域的判定规则
4. 若能力域确被触发，按 `core/03-risk-tiers.md` 的流程得出最终级别

## 常见误判

- 改了 `bridge-protocol.ts` 中的一行注释 → 文件匹配桥接协议模式，但注释变更不触及协议语义 → **不应判定为高影响**
- 改了 `gateway.ts` 中的日志级别常量 → 文件匹配网关核心模式，但日志级别不影响调度逻辑 → **不应判定为高影响**
